STON.fi Module (Under Active Development)

Socialbridge is a non-custodial TON wallet currently in active development (dev-only/private build). Our goal is to make liquidity and staking feel wallet-native: clear UX states, reliable execution, and safe resolution paths for multi-step DeFi actions.

What we are building

We are implementing a dedicated STON.fi integration module (stonfi-kit) so the STON.fi flows are:

  • Testable (integration/regression tests, fixtures)
  • Maintainable (clean interfaces and isolated dependencies)
  • Reliable (clear transaction lifecycle and predictable outcomes)

Security note: key management and signing remain inside the wallet. The module never handles secrets and only receives a generic transaction-sender interface.

Deliverables (in progress)

1) Single-sided liquidity provision (one-token entry)

Users can provide liquidity by depositing a single token (e.g., TON). The wallet orchestrates quote/simulation → user confirmation → required swap(s) → add liquidity, and shows a clear lifecycle for each step.

2) Clear transaction lifecycle UX

A consistent flow state machine is used across DeFi actions:

prepare → awaiting signature → sent → confirmed / failed / canceled

with safe retry behavior and understandable error messages.

3) In-wallet completion / refund path

Multi-step liquidity actions can sometimes end up incomplete. We are implementing a safe in-wallet path to finalize or refund without requiring users to switch to the web UI.

4) Core staking flows

Stake/unstake, staking positions display, and basic activity history in-wallet.

Milestones (high level)

  • M1 — Foundation: stonfi-kit scaffold, clean API, normalized models, error taxonomy, flow state machine skeleton
  • M2 — Single-sided liquidity: production-ready orchestration + UX lifecycle
  • M3 — Completion/refund + hardening: incomplete attempts, timeouts, rounding/slippage boundaries
  • M4 — Staking + quality: staking core flows, integration/regression tests, minimal documentation/demo
